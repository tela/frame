# Media Library

## Job

Browse and manage wardrobe, prop, and location images. These are visual reference assets used in generation (wardrobe items for clothing swap/dressed generation, locations for scene backgrounds) and in Fig's production workflows.

## Who Uses This

The user, when:
- Browsing available garments for use in studio generation or standard catalog assignment
- Adding new wardrobe/prop/location images
- Finding a specific asset for reference

## What It Shows

### Top-Level Navigation
Three tabs or sections:
- **Wardrobe** — garments, outfits, accessories
- **Props** — objects, items
- **Locations** — backgrounds, settings, environments

### Item Grid
For each content type, a grid of items:
- Primary image thumbnail
- Item name
- Image count (if multiple images per item)

### Filtering
- Search by name
- Sort by name, date added, image count

### Item Detail
When an item is selected:
- All images for this item (primary + additional)
- Item name (editable)
- Actions: add image, set primary image, remove image
- For wardrobe items: which eras this garment is assigned to as a standard catalog item

## Actions

- Browse by content type
- Search and filter
- View item detail
- Add/remove images to items
- Create new items (ID provided by Fig during registration, or generated by Frame for manually-added items)
- Assign wardrobe items to eras as standard catalog garments

## Data Requirements

- `GET /api/v1/media/{type}` — list items
- `GET /api/v1/media/{type}/{id}` — item detail with images
- `POST /api/v1/media/{type}` — create item
- `POST /api/v1/media/{type}/{id}/images` — add image
- Image serving via standard image endpoints

## Notes

- Media items are primarily registered by Fig (wardrobe, props, locations originate there). Frame may also support manual addition for assets that don't exist in Fig.
- The wardrobe section has the deepest integration with the rest of Frame — garment images are used in generation templates and standard catalog assignments.
- Props and locations are simpler — mostly image storage and serving. Their deeper production semantics live in Fig.
